# Спецификация MVP vKOROBKU

## 1. Назначение

vKOROBKU помогает освободить место на NTFS-накопителе, применяя встроенное прозрачное сжатие Windows к файлам игр. Приложение не меняет логическое содержимое файлов и не разрабатывает собственный драйвер файловой системы.

## 2. Поддерживаемая среда

- Windows 10/11 x64;
- интерфейс только на русском языке;
- Steam и ручной выбор каталога;
- тёмный игровой интерфейс;
- без фоновой службы и автоматической обработки обновлений.

## 3. Пользовательский сценарий

1. Приложение без повышения прав собирает характеристики ПК.
2. Находит Steam и читает манифесты библиотек.
3. Пользователь выбирает игру или добавляет каталог.
4. Выполняется проверка NTFS, свободного места, reparse points и доступности файлов.
5. Программа формирует стратифицированную выборку объёмом до 512 МБ, 1 ГБ или 2 ГБ в зависимости от выбранной точности.
6. Пользователь нажимает «Оптимизировать»; программа автоматически оценивает алгоритмы и выбирает подходящий.
7. Перед изменением файлов показываются краткий прогноз экономии и выбранный режим без перегрузки интерфейса техническими деталями.
8. Пользователь подтверждает операцию; повышение прав запрашивается только при необходимости.
9. Программа сохраняет анализ между запусками, ведёт журнал операции и показывает фактический результат.
10. Статус WOF/NTFS-сжатия сохраняется между запусками; для сжатой игры доступна распаковка.
11. Игру можно полностью распаковать.

## 4. Алгоритмы

- XPRESS4K — минимальная нагрузка, меньшая экономия;
- XPRESS8K — быстрый режим;
- XPRESS16K — предполагаемый режим по умолчанию;
- LZX — максимальная экономия, наибольшая стоимость декомпрессии;
- распаковка — возврат файлов к несжатому состоянию.

Выбор всегда остаётся за пользователем. Рекомендация приложения не блокирует остальные варианты.

## 5. Оценка экономии

### Выборка

Файлы группируются по диапазону размера и расширению. Выборка включает данные из разных групп и участков больших файлов, а не первые N файлов. Доступны быстрый лимит 512 МБ, точный 1 ГБ, максимальный 2 ГБ и автоматический режим. Фактический объём ограничивается примерно 5% игры; для установок до 1 ГБ используется практический минимум до 64 МБ.

Необходимо отдельно учитывать:

- файлы меньше размера кластера;
- уже сжатые, разреженные и зашифрованные файлы;
- архивы, видео и другие плохо сжимаемые форматы;
- недоступные файлы;
- reparse points, по которым нельзя рекурсивно переходить.

Выборка копируется во временный каталог на NTFS, последовательно сжимается штатным механизмом Windows, измеряется её физический размер. После каждого алгоритма временные данные удаляются.

### Результат

Показывается диапазон, а не гарантированное значение:

- текущий логический и физический размер;
- прогноз физического размера после операции;
- прогноз освобождённого места;
- доля исследованных данных;
- доверие: низкое, среднее или высокое.

## 6. Оценка производительности

Приложение не прогнозирует FPS. Оно оценивает только ввод-вывод и возможное влияние на запуск, загрузочные экраны и потоковую подгрузку.

Учитываются:

- скорость чтения исходной выборки;
- скорость чтения сжатой выборки с декомпрессией;
- производительность накопителя;
- загрузка и пропускная способность CPU;
- типы файлов в выборке.

Для каждого алгоритма показываются скорость исходной несжатой выборки, скорость прозрачного чтения после сжатия и разница в процентах. Результат также отображается категориями: «вероятно быстрее», «незаметно», «возможно медленнее», «не рекомендуется». Это оценка загрузки данных, а не FPS.

## 7. Безопасность

Перед изменением файлов обязательны:

- проверка Windows x64 и NTFS;
- проверка свободного места под временную выборку;
- обнаружение запущенной игры и предупреждение о запущенном Steam;
- запрет перехода через junction/symlink/reparse point;
- пропуск EFS-зашифрованных и недоступных файлов;
- журнал результата каждого файла или пакета;
- корректная отмена без попытки отката уже успешно обработанных файлов; worker останавливает текущий пакет и сохраняет валидное частично обработанное состояние;
- повторный анализ после прерывания;
- операция полной распаковки.

Изменение или обновление файла лаунчером может снять с него сжатие. MVP не следит за этим в фоне.

## 8. Границы MVP

Не входят: Microsoft Store/Xbox, Epic/GOG/EA/Ubisoft/Battle.net, фоновые службы, автоматическая повторная упаковка, облачная телеметрия, собственный драйвер и прогноз FPS.

## 9. Критерии готовности

- Steam-игры корректно находятся в нескольких библиотеках;
- ручная папка добавляется без обхода reparse points;
- оценка никогда не изменяет исходную игру;
- фактическая экономия на тестовом наборе попадает в показанный диапазон в большинстве тестов;
- отмена не повреждает файлы;
- распаковка возвращает читаемые исходные данные;
- приложение работает без прав администратора до защищённой операции.
